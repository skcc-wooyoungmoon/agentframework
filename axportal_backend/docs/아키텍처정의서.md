## 3. API 응답 표준 아키텍처

### 3.1 통합 응답 체계 개요

**AxportalBackend**는 모든 REST API 응답에 대해 **AxResponseEntity<PageResponse<T>>** 기반의 표준화된 응답 구조를 사용합니다.

```mermaid
graph TD
    A[Client Request] --> B[Controller Layer]
    B --> C[Service Layer]
    C --> D[Repository Layer]
    D --> E[Entity/Data]
    
    E --> F[DTO Conversion]
    F --> G{Response Type?}
    
    G -->|Page Data| H[PageResponse.from(Page)]
    G -->|Single Data| I[Entity Response DTO]
    G -->|No Data| J[Void Response]
    
    H --> K[AxResponseEntity.okPage()]
    I --> L[AxResponseEntity.ok()]
    J --> M[AxResponseEntity.ok()]
    
    K --> N[AxResponse with PageResponse]
    L --> O[AxResponse with Entity Data]
    M --> P[AxResponse with Success Message]
    
    N --> Q[Standardized JSON Response]
    O --> Q
    P --> Q
    
    Q --> R[Client Response]
```

### 3.2 응답 구조 계층

#### Layer 1: AxResponseEntity (Spring ResponseEntity 확장)
```java
AxResponseEntity<T> extends ResponseEntity<AxResponse<T>>
```

#### Layer 2: AxResponse (응답 래퍼)
```java
@Getter
@Builder
public class AxResponse<T> {
    private final boolean success;          // 성공 여부
    private final String message;           // 응답 메시지
    private final T data;                   // 응답 데이터
    private final ErrorInfo error;          // 에러 정보 (실패 시)
    private final LocalDateTime timestamp;  // 응답 시간
    private final String path;              // 요청 경로

    // 에러 정보 내부 클래스 (✅ 최신 구조)
    @Getter
    @Builder
    public static class ErrorInfo {
        private final String code;          // 에러 코드
        private final String message;       // 에러 메시지
        private final String details;       // 상세 설명
        private final LocalDateTime timestamp;
        private final String path;
        private final List<FieldError> fieldErrors; // 필드 검증 에러
    }

    // 필드 에러 내부 클래스 (✅ 최신 구조)
    @Getter
    @Builder
    public static class FieldError {
        private final String field;         // 필드명
        private final Object rejectedValue; // 거부된 값
        private final String message;       // 에러 메시지
    }
}
```

#### Layer 3: PageResponse (페이징 전용 데이터) (✅ 최신 단순화 구조)
```java
@Getter
@Builder
public class PageResponse<T> {
    private List<T> content;                // 페이지 컨텐츠 목록
    private PageableInfo pageable;          // 페이지 정보 객체
    private long totalElements;             // 전체 요소 수
    private int totalPages;                 // 전체 페이지 수
    private boolean first;                  // 첫 번째 페이지 여부
    private boolean last;                   // 마지막 페이지 여부
    private boolean hasNext;                // 다음 페이지 존재 여부
    private boolean hasPrevious;            // 이전 페이지 존재 여부
    
    // 페이지 정보 내부 클래스
    @Getter
    @Builder
    public static class PageableInfo {
        private int page;                   // 현재 페이지 번호 (0부터 시작)
        private int size;                   // 페이지 크기
        private String sort;                // 정렬 정보
    }
        private String url;                 // 링크 URL (비활성화시 null)
        private String label;               // 링크 라벨
        private boolean active;             // 활성 상태
        private int page;                   // 페이지 번호
    }
}
```

### 3.3 응답 타입별 아키텍처 패턴

#### 페이징 조회 응답 (✅ 최신 단순화 구조)
```java
// Controller Pattern (단순화)
AxResponseEntity<PageResponse<UserResponse>> getAllUsers(Pageable pageable)

// Service Pattern (PageResponse 직접 반환)
PageResponse<UserResponse> getUsersPageResponse(Pageable pageable)

// Service Pattern (Page 반환, Controller에서 변환)
Page<UserResponse> getUsersPage(Pageable pageable)

// 표준 응답 패턴
return AxResponseEntity.okPage(users, "목록 조회 성공");
```

#### 페이징 응답 생성 아키텍처
```mermaid
graph TD
    A[Page<T> 입력] --> B[PageResponse.from() 호출]
    B --> C[PageableInfo 생성]
    C --> D[페이징 메타데이터 추가]
    D --> E[PageResponse<T> 반환]
```
    
    G --> J{첫 페이지?}
    J -->|Yes| K[url: null, active: false]
    J -->|No| L[실제 URL, active: false]
    
    I --> M{마지막 페이지?}
    M -->|Yes| N[url: null, active: false]
    M -->|No| O[실제 URL, active: false]
    
    H --> P[모든 페이지 번호 링크]
    
    K --> Q[Pagination 객체 생성]
    L --> Q
    N --> Q
    O --> Q
    P --> Q
    
    Q --> R[PageResponse<T> 반환]
```

#### 단건 조회 응답
```java
// Controller Pattern
AxResponseEntity<UserResponse> getUserById(Long id)

// Service Pattern
UserResponse getUserById(Long id)
```

#### 생성/수정/삭제 응답
```java
// 생성: 생성된 엔티티 반환
AxResponseEntity<UserResponse> createUser(CreateUserRequest request)

// 수정: 수정된 엔티티 반환
AxResponseEntity<UserResponse> updateUser(Long id, UpdateUserRequest request)

// 삭제: 성공 메시지만 반환
AxResponseEntity<Void> deleteUser(Long id)
```

### 3.6 응답 표준화 장점

#### 클라이언트 관점
- **일관된 응답 구조**: 모든 API가 동일한 응답 형식 제공
- **명확한 메타데이터**: 페이징 정보, 상태 코드, 타임스탬프 등 풍부한 메타데이터
- **예측 가능한 구조**: 응답 파싱 로직의 표준화 가능
- **상세한 에러 정보**: ErrorInfo 기반 풍부한 에러 컨텍스트 제공 (✅ 최신 추가)
- **풍부한 페이징 링크**: Previous/Next/페이지번호 링크로 쉬운 네비게이션 구현 (✅ 최신 추가)

#### 백엔드 관점
- **개발 생산성**: 표준화된 패턴으로 일관된 코드 작성
- **유지보수성**: 응답 구조 변경 시 중앙화된 관리
- **확장성**: 새로운 기능 추가 시 기존 패턴 재사용
- **통합 에러 처리**: GlobalExceptionHandler 기반 중앙화된 예외 관리 (✅ 최신 추가)
- **자동 링크 생성**: baseUrl 기반 자동 페이징 링크 생성으로 개발 효율성 향상 (✅ 최신 추가)

#### 아키텍처 관점
- **계층 분리**: 각 계층의 책임이 명확하게 분리
- **재사용성**: PageResponse, AxResponse 등 공통 컴포넌트 재사용
- **테스트 용이성**: 표준화된 구조로 테스트 작성 간소화
- **에러 추적성**: 요청 경로 및 타임스탬프 기반 에러 추적 지원 (✅ 최신 추가)

### 4.2 각 계층의 책임

#### 4.2.1 Presentation Layer (표현 계층)
- **Controllers**: REST API 엔드포인트 정의, HTTP 요청/응답 처리
- **Exception Handlers**: 글로벌 예외 처리, 에러 응답 표준화
- **DTOs**: 외부와의 데이터 교환을 위한 객체
- **OpenAPI**: API 문서 자동 생성 및 Swagger UI 제공

#### 4.2.2 Application Layer (애플리케이션 계층)
- **Services**: 비즈니스 로직 구현, 트랜잭션 관리
- **Configuration**: 애플리케이션 설정 관리
- **Security**: 인증/인가 로직 구현
- **Validation**: 입력값 검증 로직

#### 4.2.3 Domain Layer (도메인 계층)
- **Entities**: 비즈니스 도메인 객체, JPA 엔티티
- **Domain Services**: 도메인별 핵심 비즈니스 로직
- **Business Rules**: 비즈니스 규칙 및 제약사항
- **Mappers**: 도메인 객체와 DTO 간의 변환

#### 4.2.4 Infrastructure Layer (인프라 계층)
- **Repositories**: 데이터 액세스 추상화
- **Feign Clients**: 외부 API 연동 인터페이스
- **Database Configuration**: 데이터베이스 설정 및 연결
- **External Integration**: 외부 시스템 연동 구현

### 6.4 데이터베이스 설계 원칙

#### 6.4.1 명명 규칙
- **테이블명**: snake_case, 복수형 (예: users, user_roles, sample_users)
- **컬럼명**: snake_case (예: created_at, user_id, email_address)
- **인덱스명**: idx_테이블명_컬럼명 (예: idx_users_email, idx_users_username)
- **제약조건명**: 타입_테이블명_컬럼명 (예: uk_users_email, fk_user_roles_user_id)

#### 6.4.2 공통 필드
모든 엔티티는 BaseEntity를 상속받아 다음 공통 필드를 포함:
- `created_at`: 생성일시 (자동 설정, 수정 불가)
- `updated_at`: 수정일시 (자동 갱신)
- `created_by`: 생성자 (추후 Spring Security Context에서 자동 설정)
- `updated_by`: 수정자 (추후 Spring Security Context에서 자동 설정)

#### 6.4.3 데이터 타입 표준
- **ID**: BIGINT (Long) - 자동 증가
- **문자열**: VARCHAR (길이 명시)
- **날짜시간**: TIMESTAMP (LocalDateTime)
- **불린**: BOOLEAN (Boolean)
- **열거형**: VARCHAR (Enum.name())
- **대용량 텍스트**: TEXT 또는 CLOB

@RestController
@RequestMapping("/api/v1/sample/users")
@RequiredArgsConstructor
@Tag(name = "Sample User", description = "샘플 사용자 관리 API")
public class SampleUserController {

    private final SampleUserService sampleUserService;

    @PostMapping
    @Operation(summary = "사용자 생성", description = "새로운 사용자를 생성합니다.")
    @ApiResponses({
        @ApiResponse(responseCode = "201", description = "사용자 생성 성공"),
        @ApiResponse(responseCode = "400", description = "잘못된 요청 데이터"),
        @ApiResponse(responseCode = "409", description = "사용자명 또는 이메일 중복")
    })
    public AxResponseEntity<SampleUserRes> createUser(
            @Valid @RequestBody SampleUserCreateReq request) {
        
        SampleUserRes user = sampleUserService.createUser(request);
        return AxResponseEntity.created(user, "사용자가 성공적으로 생성되었습니다.");
    }

    @GetMapping("/{id}")
    @Operation(summary = "사용자 조회", description = "ID로 사용자 정보를 조회합니다.")
    public AxResponseEntity<SampleUserRes> getUserById(@PathVariable Long id) {
        SampleUserRes user = sampleUserService.getUserById(id);
        return AxResponseEntity.ok(user, "사용자 조회 성공");
    }

    @GetMapping
    @Operation(summary = "사용자 목록 조회", description = "전체 사용자 목록을 페이징으로 조회합니다.")
    public AxResponseEntity<Page<SampleUserRes>> getAllUsers(
            @PageableDefault(size = 20, sort = "createdAt", direction = Sort.Direction.DESC) 
            Pageable pageable) {
        
        Page<SampleUserRes> users = sampleUserService.getAllUsers(pageable);
        return AxResponseEntity.ok(users, "사용자 목록 조회 성공");
    }
}
